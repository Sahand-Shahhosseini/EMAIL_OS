# 01 Canonical Objects

Date: 2026-03-27
Status: Bootstrap scaffold
Purpose: record the minimum canonical object families of `EMAIL_OS`.

## Core object families
- email record
- stable message identity
- rule
- triage state
- classification
- priority
- due/follow-up state
- sync state
- backfill state
- report artifact

## Boundary rule
These objects define the Email-OS family only as an email-operational system.
They do not replace generic project registry, DB-core doctrine, or account-wide routing objects.
